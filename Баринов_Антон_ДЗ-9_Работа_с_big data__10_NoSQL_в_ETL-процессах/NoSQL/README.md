# ДЗ 10 - NoSQL в ETL-процессах

## Цель
1. Подготовить тестовые данные.
2. Подготовить и активировать трансфер.
3. Проверить работоспособность трансфера.

## Задача
Перенести данные из Kafka в MongoDB с помощью Data Transfer.

## Данные
Использованы тестовые данные sample.json из документации Yandex Cloud.

## Шаги решения
1. Развертывание кластера-источника Managed Service for Kafka, создание топика sensors и пользователя mkf-user с правами продюсера и консюмера
2. Развертывание кластера-приёмника Yandex StoreDoc (MongoDB), создание базы данных db1 и пользователя mmg-user
3. Настройка групп безопасности для разрешения трафика между Kafka, MongoDB и Data Transfer
4. Генерация тестовых данных в формате JSON (данные автомобильных сенсоров) и их отправка в топик sensors через kcat
5. Создание эндпоинтов Data Transfer: источник (Kafka с JSON-схемой) и приемник (MongoDB)
6. Создание и активация трансфера типа "Репликация", проверка статуса и переноса данных в коллекцию sensors базы db1
7. Проверка результата через mongosh: чтение документов из MongoDB командой `db.sensors.find()`

## Результат
Данные успешно прочитаны.

## Технологии
- Kafka, MongoDB
- Managed Service for Kafka
- Yandex StoreDoc
- Data Transfer
